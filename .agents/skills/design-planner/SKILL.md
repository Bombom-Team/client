# Design Planner (Orchestrator)

`design-planner` 에이전트와 사용자 사이의 Q&A 루프를 오케스트레이션합니다. 에이전트가 봄봄 디자인 가이드를 읽고 설계를 담당하며, 이 스킬이 대화를 담당합니다.

## 기준 문서

`.claude/skills/design-planner/bombom-design-guide.md` — 봄봄 서비스의 토큰·컴포넌트·레이아웃 기준. 에이전트가 매 호출마다 이 문서를 읽어서 참조합니다. 가이드에 없는 규칙/컴포넌트를 지어내지 않습니다.

가이드가 오래됐다고 판단되면 사용자에게 알리고 갱신 제안. 자동으로 덮어쓰지 않음.

## 플로우

```
사용자 아이디어/PRD
    ↓
[1] design-planner 에이전트 호출 (1회차: 초안 + 질문)
    ↓
에이전트 응답: "❓ 사용자 확인 필요" + 초안
    ↓
[2] AskUserQuestion으로 사용자에게 질문 (2~3개)
    ↓
사용자 답변
    ↓
[3] design-planner 에이전트 재호출 (2회차: 원본 + 사용자 답변)
    ↓
에이전트 응답: 최종 디자인 설계 (답변 반영, 마커 포함)
    ↓
사용자에게 출력
```

## 단계별 가이드

### 1. 에이전트 1회차 호출

`Agent` 툴을 `subagent_type: "design-planner"`로 호출합니다. 프롬프트:

```
아이디어: <사용자가 설명한 내용 또는 PRD 요약 그대로>

먼저 `.claude/skills/design-planner/bombom-design-guide.md`를 Read로 읽으세요. 그 가이드의 토큰·컴포넌트만 사용해 위 아이디어의 화면 구성 + 컴포넌트 목록 + 인터랙션 명세를 작성해주세요. 불명확한 지점은 `❓ 사용자 확인 필요` 섹션에 2~3개로 정리해주세요.
```

### 2. 질문 파싱 & 사용자에게 전달

에이전트 응답의 `❓ 사용자 확인 필요` 섹션에서 질문을 추출합니다. 예:

```markdown
## ❓ 사용자 확인 필요

1. 신규 페이지로 만들지, 기존 아티클 상세 안에 섹션으로 넣을지?
2. 답변 제출 UI는 모달/바텀시트/인라인 중 어느 것?
3. 비로그인 유저도 입력 가능 여부?
```

→ `AskUserQuestion` 툴로 전달. 각 질문은 별도 question 항목으로, 객관식 선택지를 만들기 어려우면 "Other"로 자유 입력.

**선택지 작성 팁**:
- 화면 배치 질문이면 흔한 선택지(신규 페이지 / 기존 페이지 내 섹션 / 모달)를 옵션으로
- 명확한 yes/no 질문이면 그대로 2개 옵션
- Option label은 짧게 (1~5 단어)

### 3. 에이전트 2회차 호출

수집한 답변을 정리해 재호출합니다. 프롬프트:

```
아이디어: <원본 아이디어/PRD>

사용자 답변:
- Q: <질문 1>
  A: <사용자 답변 1>
- Q: <질문 2>
  A: <사용자 답변 2>
...

먼저 `.claude/skills/design-planner/bombom-design-guide.md`를 Read로 읽으세요. 답변을 반영해 최종 디자인 설계를 작성해주세요. 사용자 답변에서 가져온 내용은 인라인 마커(`_(사용자 답변)_` 또는 `_(사용자 답변 기반: "...")_`)로 출처를 표시해주세요.
```

### 4. 사용자에게 최종 출력

에이전트가 돌려준 마크다운을 그대로 전달합니다. 앞뒤에 짧은 안내만:

```
디자인 설계 초안입니다. 디자이너/개발자와 검토하세요.

<에이전트 출력 그대로>
```

출력을 fenced code block으로 감싸지 않습니다(복사 불편).

## 엣지 케이스

- **사용자가 PRD·문맥을 충분히 제공한 경우**: 에이전트가 1회차에서 `❓ 사용자 확인 필요` 섹션 없이 바로 완성본을 낼 수 있음. 이 경우 2회차 생략.
- **사용자가 일부 질문만 답한 경우**: 답변 있는 것만 에이전트에 전달. 나머지는 `[TODO: 확인 필요]`로 남음.
- **사용자가 질문 거부 ("그냥 써줘")**: 2회차 생략, 1회차 초안 그대로 출력.
- **봄봄 외 서비스 디자인 요청**: `bombom-design-guide.md`는 봄봄 전용이므로 "이 스킬은 봄봄 기능 디자인용"이라고 안내하고 사용자에게 다른 서비스인지 확인.
- **에이전트 호출 실패**: 새 에이전트가 세션 재시작 전에는 `subagent_type`으로 안 잡힐 수 있음. 이때는 "새 세션에서 다시 시도해주세요" 안내.

## 금지

- 에이전트를 건너뛰고 스킬 단독으로 디자인 설계 작성 (가이드 참조 책임이 에이전트에 있음)
- 한 번에 4개 이상 질문 던지기 (피로)
- 에이전트 출력을 재해석·요약해서 마커 제거 (출처가 사라짐)
- Figma MCP 자동 호출 (사용자가 명시적으로 요청할 때만 별도 플로우)
- tsx/css 구현 코드 생성 (구현은 Plan/Edit 스킬 영역)
