# Update Context

서비스 컨텍스트 문서의 변경 잦은 두 섹션만 안전하게 갱신한다.

## 대상 파일

`docs/ai-context/product-context.md`

수정 허용 섹션:
- `핵심 기능 맵`
- `주요 화면 구조`

나머지 섹션은 수정하지 않는다.

## 플로우

1. 사용자에게 "무엇이 출시/변경됐는지" 확인한다.
2. 영향 섹션을 확인한다: 기능 맵 / 화면 구조 / 둘 다.
3. 위 두 섹션에서 필요한 행/항목만 수정한다.
4. 각 섹션의 `_마지막 업데이트: YYYY-MM-DD_`를 오늘 날짜로 갱신한다.
5. 변경 요약을 1~3줄로 보고한다.

## 수정 규칙

- 기능 추가: 표 행 추가 (`기능 | 설명 | 연결되는 가치`).
- 기능 제거/개편: 기존 행 삭제 또는 설명 수정.
- 화면 추가/이동: `현재 존재`와 `추후 추가 예정` 사이에서 항목 이동/추가.
- 연결되는 가치 번호(1~5)가 불명확하면 반드시 사용자에게 확인한다.

## 금지

- 출시되지 않은 기획을 반영하지 않는다.
- 대상 두 섹션 외 문단(가치 명제/계정 모델 등)을 수정하지 않는다.
- 가치 번호를 추측하지 않는다.
