# Git 커밋 메시지 컨벤션

봄봄(BomBom) 프로젝트의 커밋 메시지는 항상 다음 형식을 따른다. 커밋 작업 시(커밋 메시지를 작성하거나 검토할 때) 반드시 이 문서를 참고한다.

## 형식

```
type: 한국어 명사형 설명
```

규칙:

- `type` 뒤에 콜론(`:`) + 공백 하나.
- **scope는 쓰지 않는다.** (`type(scope): ...` 형태 금지)
- **티켓 ID prefix는 제목에 넣지 않는다.** (`[BOM-XXXX]` 같은 prefix 금지 — PR 제목 형식에는 들어가지만 커밋 제목에는 들어가지 않는다.)
- **어미는 명사형**으로 끝맺는다 ("추가", "수정", "변경", "보완", "제거" 등). "~해요" 같은 존댓말 어미는 쓰지 않는다.
- **마침표 없음.**
- **본문(body)은 기본적으로 쓰지 않는다.** 변경 의도가 제목만으로 충분히 안 보이는 경우에만 한 단락 추가한다.
- **`Co-Authored-By:` 트레일러는 사용하지 않는다.** 봄봄 컨벤션상 첨부하지 않는다.

## 사용 가능한 type

| type       | 사용처                                                |
| ---------- | ----------------------------------------------------- |
| `feat`     | 새로운 기능 추가                                      |
| `fix`      | 버그 수정                                             |
| `docs`     | 문서 수정 (코드 변경 없음)                            |
| `style`    | 코드 포맷팅, 세미콜론 등 스타일 변경 (논리 변경 없음) |
| `design`   | 사용자 UI 디자인 변경 (CSS, 레이아웃 등)              |
| `refactor` | 리팩토링 (기능 변화 없음)                             |
| `test`     | 테스트 관련 코드 추가/수정                            |
| `chore`    | 빌드, 패키지 매니저 설정 등 기타 작업                 |

## 예시

```
feat: 로그인 기능 추가
fix: 비밀번호 검증 오류 수정
docs: README 업데이트
style: 코드 포맷팅 적용 (세미콜론 추가 등)
design: 스위칭 헤더의 로고 크기 조정
refactor: 중복 코드 제거
test: 로그인 테스트 코드 추가
chore: 빌드 설정 변경
```
