# Product Planner (Orchestrator)

`product-planner` 에이전트와 사용자 사이의 Q&A 루프를 오케스트레이션합니다. 에이전트가 순수 작성을 맡고, 이 스킬이 대화를 담당합니다.

## 플로우

```
사용자 아이디어
    ↓
[1] product-planner 에이전트 호출 (1회차: 초안 + 질문)
    ↓
에이전트 응답: "❓ 사용자 확인 필요" + 초안
    ↓
[2] AskUserQuestion으로 사용자에게 질문 (2~3개)
    ↓
사용자 답변
    ↓
[3] product-planner 에이전트 재호출 (2회차: 원본 아이디어 + 사용자 답변)
    ↓
에이전트 응답: 최종 PRD (사용자 답변 반영, 마커 포함)
    ↓
사용자에게 출력
```

## 단계별 가이드

### 1. 에이전트 1회차 호출

`Agent` 툴을 `subagent_type: "product-planner"`로 호출합니다. 프롬프트:

```
아이디어: <사용자가 설명한 내용 그대로>

위 아이디어를 PRD + 이슈 분해로 작성해주세요. 불명확한 지점은 `❓ 사용자 확인 필요` 섹션에 2~3개로 정리해주세요.
```

### 2. 질문 파싱 & 사용자에게 전달

에이전트 응답의 `❓ 사용자 확인 필요` 섹션에서 질문을 추출합니다. 예:

```markdown
## ❓ 사용자 확인 필요

1. 이 기획을 트리거한 배경이 뭔가요? (사용자 피드백 / 지표 / 기획 요청)
2. 답변 제출을 강제할지, 건너뛰기 허용할지?
3. 비로그인 유저도 답변 저장할지?
```

→ `AskUserQuestion` 툴로 그대로 전달합니다. 각 질문은 별도 question 항목으로, 객관식 선택지를 만들기 어려우면 "Other"로 자유 입력 유도.

**선택지 작성 팁**:
- 에이전트 질문이 열린 질문이면 흔한 답변 2~3개를 options로, 나머지는 Other로
- 명확한 yes/no 질문이면 그대로 2개 options
- Option labels는 짧게 (1~5 단어)

### 3. 에이전트 2회차 호출

수집한 답변을 정리해 재호출합니다. 프롬프트:

```
아이디어: <원본 아이디어>

사용자 답변:
- Q: <질문 1>
  A: <사용자 답변 1>
- Q: <질문 2>
  A: <사용자 답변 2>
...

위 정보를 반영해 최종 PRD + 이슈 분해를 작성해주세요. 사용자 답변에서 가져온 내용은 인라인 마커(`_(사용자 답변)_` 또는 `_(사용자 답변 기반: "...")_`)로 출처를 표시해주세요.
```

### 4. 사용자에게 최종 출력

에이전트가 돌려준 마크다운을 그대로 전달합니다. 앞뒤에 짧은 안내만 추가:

```
기획안 초안입니다. 검토 후 Linear에 넣으세요.

<에이전트 출력 그대로>
```

출력을 fenced code block으로 감싸지 않습니다(복사 불편).

## 엣지 케이스

- **사용자가 충분한 정보를 이미 제공한 경우**: 에이전트가 1회차에서 `❓ 사용자 확인 필요` 섹션 없이 바로 완성본을 낼 수 있음. 이 경우 2회차 생략하고 바로 출력.
- **사용자가 일부 질문만 답한 경우** (AskUserQuestion에서 Other로 모호하게 답 / 건너뜀): 답변 있는 것만 에이전트에 전달. 나머지는 에이전트가 알아서 `[TODO]`로 남김.
- **사용자가 질문 자체를 거부** (e.g. "그냥 대충 써줘"): 2회차를 생략하고 1회차 초안을 그대로 출력.
- **에이전트 호출이 실패하는 경우**: 새로 만든 에이전트는 세션 재시작 전에는 `Agent` 툴의 `subagent_type`으로 안 잡힐 수 있음. 이때는 사용자에게 "새 세션에서 다시 시도해주세요" 안내.

## 금지

- 에이전트를 건너뛰고 스킬 단독으로 PRD 작성 (에이전트의 설계를 무력화)
- 한 번에 4개 이상 질문 던지기 (피로)
- 에이전트 출력을 재해석·요약해서 마커 제거 (출처가 사라짐)
- Linear에 이슈 자동 생성 (마크다운 출력만)
